📦 CSS Display & Box Model 深度演示

以下演示展示了 display 属性如何从根本上改变元素的盒模型行为和布局方式。

1. 基础显示模式 (Block vs Inline vs Inline-Block)

所有子元素都设置了 width: 150px; height: 80px; 和 margin: 5px;。观察它们的差异。

A. display: block (块级)
Box 1 (Block)
Box 2 (Block)
Box 3 (Block)

现象： 每个盒子独占一行，无视宽度限制（除非设了 width），宽高设置生效。

B. display: inline (行内)
Box 1 (Inline)
Box 2 (Inline)
Box 3 (Inline)

现象： 盒子排在一行，宽高设置失效 (由内容决定大小)，margin-top/bottom 失效。它们像文字一样流动。

C. display: inline-block (行内块)
Box 1 (IB)
Box 2 (IB)
Box 3 (IB)

现象： 盒子排在一行 (不换行)，但宽高设置生效！这是制作按钮、卡片的完美选择。

现象： 中间那个包裹层（原本可能有背景色或边框）“消失”了，绿色的子元素直接受最外层红色 Grid 容器的控制。用于解决嵌套过深导致的布局问题。

C. display: flow-root (现代 Clearfix)
❌ 没有 flow-root (高度塌陷):
Float Left

文字环绕... 父元素灰色背景没有包住红色的浮动块，高度塌陷了！

✅ 加上 flow-root (自动清除):
Float Left

文字环绕... 父元素灰色背景现在完美包住了红色的浮动块！不需要伪元素 clearfix。

现象： `flow-root` 创建了一个新的块级格式化上下文 (BFC)，自动包含内部的浮动元素，是现代清除浮动的最佳实践。
